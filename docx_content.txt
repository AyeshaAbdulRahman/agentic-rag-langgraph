1.  Project Overview
NeuroCare AI is a domain-restricted Agentic RAG chatbot for dementia care. It is built with LangGraph and uses a multi-node intelligent pipeline to ensure safe, accurate, and emotionally aware responses.
What the chatbot does:
Answers ONLY dementia and Alzheimer-related questions — off-topic questions are blocked with a kind message
Corrects ALL spelling mistakes in user query automatically before processing (any word, not just medical terms)
Detects the emotional tone of the user's message (anxious, sad, frustrated, calm, confused)
Retrieves the most relevant chunks from your uploaded PDF/TXT documents using FAISS vector search
Grades each retrieved chunk for relevance — filters out irrelevant content before answering
Falls back to free DuckDuckGo web search if document data is insufficient
Generates a grounded, accurate answer and checks it for hallucinations
Responds with emotionally appropriate language based on detected user tone
Shows source documents and page numbers for every answer (transparency)
Supports both Mistral (free) and OpenAI as the LLM — switchable via .env
2.  System Architecture — LangGraph Agent Flow
The system is a directed graph. Each node does exactly one job. The graph decides which path to take based on the results of each node.
FULL FLOW:

USER MESSAGE
    ↓
[Node 1]  Spell Check Node         → Fixes ALL typos automatically
    ↓
[Node 2]  Tone Detection Node      → Detects: anxious / sad / frustrated / calm / confused
    ↓
[Node 3]  Topic Guard Node         → Is this dementia-related? NO → END with kind refusal
    ↓ YES
[Node 4]  Retrieve Node            → FAISS search → top 5 chunks from your documents
    ↓
[Node 5]  Grade Documents Node     → LLM grades each chunk: relevant or not?
    ↓
  [Decision] Enough relevant docs?
    ↓ YES                              ↓ NO
[Node 6a] Generate Node            [Node 6b] Web Search Node (DuckDuckGo free)
    ↓                                  ↓
[Node 7]  Grade Answer Node        → Is answer grounded? NO → retry generate once
    ↓ YES
[Node 8]  Emotional Response Node  → Wraps answer in tone matching user emotion
    ↓
FINAL REPLY  (answer + sources + tone used + web_search flag)

8. Emotional Response

4.4  llm_factory.py — LLM Selector
Libraries used: langchain_mistralai (ChatMistralAI), langchain_openai (ChatOpenAI), config, dotenv
Purpose: Single function get_llm() that returns the correct LLM based on LLM_PROVIDER in .env. All 8 nodes call this function — no LLM config is repeated anywhere.
Logic:
If LLM_PROVIDER == 'mistral'  →  return ChatMistralAI(model=MISTRAL_MODEL, temperature=0)
If LLM_PROVIDER == 'openai'   →  return ChatOpenAI(model=OPENAI_MODEL, temperature=0)
Else                          →  raise clear error message

4.7  graph/nodes/spell_check.py — Node 1
Libraries used: pyspellchecker (SpellChecker), re, config
Purpose: Automatically corrects ALL spelling mistakes in the user's raw query. This runs before any other processing so all downstream nodes get clean text.
How it works:
Create SpellChecker instance
Tokenize the query into individual words using regex
For each word: call spell.correction(word) to get the best corrected version
Preserve capitalization of original word
Reconstruct the corrected sentence
Store in state.corrected_question
Important: pyspellchecker handles ALL words automatically — medical, common, and everything else.
No manual word lists needed. If a word is unknown to the dictionary it is left as-is.
Example corrections:
'demensia symtoms' → 'dementia symptoms'
'wat is alzeimer desease' → 'what is alzheimer disease'
'my granmather is forgetfull' → 'my grandmother is forgetful'
4.8  graph/nodes/tone_detect.py — Node 2
Libraries used: langchain (PromptTemplate, LLMChain), llm_factory, config
Purpose: Reads the corrected user message and detects the emotional tone. This tone is stored in state and used later by the emotional.py node to craft the right response.
Prompt sent to LLM:
Read this message from a dementia caregiver or patient:
'{message}'
Classify the emotional tone into EXACTLY ONE of these words:
anxious | sad | frustrated | confused | calm
Reply with ONLY the single word. Nothing else.
Tone meanings and how the chatbot responds:
Detected Tone

Chatbot Response Style

sad

Validating, practical, 'This is incredibly hard...'
confused

4.9  graph/nodes/topic_guard.py — Node 3
Libraries used: langchain (PromptTemplate, LLMChain), llm_factory
Purpose: Checks if the corrected query is about dementia, Alzheimer's, memory care, or caregiving. If not — ends the conversation with a kind, empathetic refusal.
Logic:
Send query to LLM with strict classification prompt
LLM replies 'YES' or 'NO' only
If YES → state.is_on_topic = True → graph continues to retrieve node
If NO  → state.is_on_topic = False → state.refusal_message is set → graph goes to END
Refusal message example:
'I'm here specifically to support with dementia-related questions.
 I'm not able to help with that topic, but I'm ready to
 assist you with anything about memory care, caregiving,
 or Alzheimer's whenever you need. 💙'
4.10  graph/nodes/retrieve.py — Node 4
Libraries used: langchain_community.vectorstores (FAISS), sentence_transformers (SentenceTransformerEmbeddings), langchain.schema (Document), config
Purpose: Loads the FAISS vector index from disk and searches for the top-K most semantically similar chunks to the corrected user question.
Steps:
Load SentenceTransformerEmbeddings (same model used in ingestion — must match)
Load FAISS index from vectorstore/ using FAISS.load_local()
Run similarity_search(state.corrected_question, k=TOP_K_RETRIEVAL)
Store results in state.documents (list of Document objects)
Each Document object contains:
page_content — the actual chunk text
metadata.source — filename (e.g., dementia_guide.pdf)
metadata.page — page number
metadata.chunk_id — unique ID
4.11  graph/nodes/grade_documents.py — Node 5
Libraries used: langchain (PromptTemplate), pydantic (BaseModel), llm_factory
Purpose: For each retrieved chunk, asks the LLM whether it is actually relevant to the user question. Removes irrelevant chunks before generation.
Grading prompt per chunk:
Question: {question}
Document chunk: {chunk_text}
Is this chunk relevant to answering the question?
Reply with ONLY 'yes' or 'no'.
Decision after grading:
If filtered_documents has 1 or more chunks → go to generate node
If filtered_documents is empty            → go to web_search node
4.12  graph/nodes/web_search.py — Node 6b
Libraries used: duckduckgo_search (DDGS), langchain_community.tools (DuckDuckGoSearchRun)
Purpose: When document retrieval yields no relevant results, searches the web using DuckDuckGo. Completely free — no API key needed.
Steps:
Use DuckDuckGoSearchRun tool from langchain_community
Search query = state.corrected_question
Fetch top 3 results
Combine results into a single context string
Store in state.web_search_results
Set state.used_web_search = True
4.13  graph/nodes/generate.py — Node 6a
Libraries used: langchain (PromptTemplate, LLMChain), llm_factory, config
Purpose: Generates the actual answer using either document chunks or web search results as context. Uses a carefully designed prompt to ensure accuracy and dementia-specific focus.
Context selection logic:
If state.filtered_documents is not empty → use document chunks as context
If state.web_search_results is set       → use web results as context
Prompt template:
You are a compassionate dementia care expert assistant.
Answer the question using ONLY the provided context.
If the context does not contain enough information, say so honestly.
Do not make up any information.
Context:
{context}
Question: {question}
Provide a clear, accurate, helpful answer:
Output: state.generation = raw answer text, state.sources = list of source references
4.14  graph/nodes/grade_answer.py — Node 7
Libraries used: langchain (PromptTemplate), llm_factory
Purpose: Anti-hallucination check. Verifies that the generated answer is actually supported by the context that was provided. Prevents the LLM from making up information.
Grading prompt:
Context provided: {context}
Generated answer: {answer}
Is this answer fully supported by the context above?
Reply with ONLY 'yes' or 'no'.
Decision:
If 'yes' → state.answer_grounded = True → go to emotional node
If 'no' and retry_count < 2 → increment retry_count → go back to generate node
If 'no' and retry_count >= 2 → accept answer anyway → go to emotional node
4.15  graph/nodes/emotional.py — Node 8
Libraries used: langchain (PromptTemplate, LLMChain), llm_factory
Purpose: The most important node for user experience. Rewrites the factual answer with the appropriate emotional tone based on what tone_detect.py detected.
Prompt template:
You are a warm, compassionate dementia care assistant.
The user's emotional tone is: {user_tone}
Rewrite this answer with the appropriate emotional awareness:
- If tone is 'anxious':    Start with reassurance. Use calm, steady language.
- If tone is 'sad':        Start with empathy. Acknowledge their pain first.
- If tone is 'frustrated': Validate their struggle. Be practical and brief.
- If tone is 'confused':   Use simple words. Break into clear steps.
- If tone is 'calm':       Be warm but professional and informative.
Original answer: {answer}
Emotionally aware response:
Example transformations:
User Tone
Original Answer
After Emotional Node
anxious
Early dementia symptoms include memory loss and confusion.
I understand how worrying this must feel. You are not alone in this. Early dementia symptoms include memory loss and confusion — and knowing this is the first step toward getting the right support. 💙
sad
Caregivers should take regular breaks to avoid burnout.
It's okay to feel overwhelmed — caring for someone with dementia is one of the hardest things a person can do. Please remember that taking regular breaks isn't selfish — it's necessary and you deserve that care too.
confused
Dementia has several stages with different characteristics.
Let me explain this simply. Dementia happens in stages. In the early stage, people may forget small things. In the middle stage, they need more help. In the later stage, they need full-time care.
4.16  graph/graph.py — LangGraph Builder
Libraries used: langgraph (StateGraph, END), all node files, state.py
Purpose: Assembles all 8 nodes into the complete LangGraph directed graph with correct conditional edges.
Graph structure:
START → spell_check_node
spell_check_node → tone_detect_node
tone_detect_node → topic_guard_node
topic_guard_node → [CONDITIONAL]
if is_on_topic == False  → END
if is_on_topic == True   → retrieve_node
retrieve_node → grade_documents_node
grade_documents_node → [CONDITIONAL]
if filtered_documents empty → web_search_node
if filtered_documents exist → generate_node
web_search_node → generate_node
generate_node → grade_answer_node
grade_answer_node → [CONDITIONAL]
if not grounded and retry < 2 → generate_node
if grounded or retry >= 2     → emotional_node
emotional_node → END
